# /나의에이전트하네스 — 인터뷰형 자동화 워크플로 설계

## 개요

본인이 매주·매일 반복하는 업무 한 가지를 골라, 그것을 **자동화 워크플로 4단계 (트리거 → 처리 → 검증 → 저장)** 로 풀어내는 대화형 스킬. 빈 종이에서 묻지 않고 한 질문씩 던져 끌어낸다. 마지막에 정리된 4단계를 `/paperbanana`로 넘겨 가로 흐름도 1장을 자동 생성한다.

> 워크숍 4교시 §2 메인 실습 도구. **"구상까지"** — 실제 자동화 구현은 강의 후 7일 챌린지 단계.

## 사용 시점

- 본인 업무 중 **매주 같은 손동작이 반복되는 것** 한 가지를 자동화 후보로 떠올렸을 때
- "에이전트로 만들고 싶긴 한데 어디서부터 손대야 할지 모르겠다"는 막막함
- 4교시 §2에서 강사 시범 직후 본인 차례

## 인터뷰 흐름 (사용자에게 한 번에 한 질문씩)

먼저 사용자에게 안내 한 줄을 출력한다:

> **나의 에이전트 하네스 엔지니어링** — 본인이 매주 반복하는 업무 한 가지를 골라, 6개 질문에 답하면 자동화 워크플로 4단계 도식 한 장이 떨어집니다. 답이 막히면 "건너뛰기" 또는 "예시 보여줘"라고 하세요.

그 다음 한 번에 한 질문만 던진다. 사용자 답을 받으면 짧게 요약 확인 ("→ 정리하면 ...") 후 다음 질문.

### Q1. 후보 업무 (한 줄)
> "본인이 매주·매일 반복하는 업무 한 가지를 한 줄로 적어 주세요. (예: '매주 금요일 학급 안내문 작성', '매월 경비 보고서 정리', '매일 오전 메일 분류')"

### Q2. 트리거 — 무엇이 시작 신호인가?
> "이 업무는 **무엇이 도착하면 / 무슨 시간이 되면 / 누가 부탁하면** 시작되나요? 4가지 보기 중 가까운 것 또는 본인 말로:
> - (a) 시간 — 매주 OO요일 OO시
> - (b) 파일·데이터 도착 — 누가 메일/카톡으로 자료를 보내옴
> - (c) 누군가의 부탁 — 동료·관리자·학부모의 요청
> - (d) 본인이 직접 시작 — 손으로 클로드 코드 열고 한 줄 입력"

### Q3. 처리 — 가운데에 무엇이 일어나나?
> "트리거가 들어오면 본인은 평소 어떤 손동작을 하나요? **2~4단계**로 끊어 적어 주세요. (예: 1) hwp 열기 → 2) 작년 안내문 복붙 → 3) 날짜·내용 수정 → 4) 학년부장 검토)"

답이 두루뭉술하면 "1단계만 더 자세히" 한 번 되묻는다.

### Q4. 검증 — 결과가 잘 나왔는지 어떻게 아나?
> "결과가 잘 나왔다고 어떻게 판단하시나요? 4가지 보기:
> - (a) 본인 눈으로 한 번 훑어본다
> - (b) 동료·관리자가 검토 후 OK
> - (c) 자동 체크 (파일 존재 확인, 글자수, 형식)
> - (d) 결과를 미리 보고 마음에 안 들면 다시"
>
> **검증이 없는 자동화는 위험합니다** — 클로드가 잘못된 결과를 만들어도 본인이 모릅니다. 어떤 식의 검증이라도 한 줄 적어 주세요.

### Q5. 저장 — 결과는 어디로 가나?
> "끝난 결과물은 어디로 가나요? (예: 학교 폴더 / 카톡 학부모방 / 노션 / 종이 인쇄 / 학년 공유 폴더 / 그냥 본인 PC)"

### Q6. 하네스 4축 매핑 — 어디에 얹을까?
> "방금 정리한 워크플로는 강사 셋팅의 4축 중 어디에 얹히면 좋을까요? 직감으로:
> - ① **CLAUDE.md** — 매번 따라붙는 지침으로 (예: '안내문은 학년부장 검토 후만 발송')
> - ② **settings.json + hooks** — 자동 실행 (예: 금요일 14시 자동 시작 — Windows 스케줄러 + `claude -p`)
> - ③ **skills/** — `/안내문` 같은 스킬 한 장 (자연어 한 줄로 자동 발동)
> - ④ **MCP** — 노션·구글·슬랙 같은 외부 서비스 연결
>
> 한 개만 골라도 되고, 여러 축 조합도 됩니다."

## 정리 단계

6개 답을 모두 받으면, 다음 형식으로 한 번 보여 주고 사용자 OK를 받는다:

```
## 본인 에이전트 하네스 — <업무명>

| 단계 | 내용 |
|------|------|
| ① 트리거 | <Q2 답> |
| ② 처리 | <Q3 답 — 2~4단계> |
| ③ 검증 | <Q4 답> |
| ④ 저장 | <Q5 답> |

**얹을 자리**: <Q6 답 — 4축 중>
```

사용자가 "이대로 도식 만들어 줘"라고 답하면 다음 단계.

## paperbanana 호출 (도식 자동 생성)

다음 프롬프트 템플릿으로 `/paperbanana`를 호출한다. `{{...}}` 자리는 인터뷰 답으로 치환:

```
제목: {{업무명}} 자동화 워크플로
구성: 4단계 가로 흐름도 (좌→우)

박스 1 (트리거): {{Q2 답을 한 줄로}}
박스 2 (처리): {{Q3 답을 한 줄로}}
박스 3 (검증): {{Q4 답을 한 줄로}}
박스 4 (저장): {{Q5 답을 한 줄로}}

박스 사이를 굵은 화살표로 연결.
검증 박스에서 처리 박스로 가는 점선 피드백 화살표 1개 추가.

색상: 짙은 청록 #0F3D44 박스 + 크림 #F5F2ED 배경 + 주황 #E08A4E 화살표.
폰트: Pretendard 한글, 박스 안 라벨은 굵게.
스타일: 학술 다이어그램 깔끔한 벡터, 흰 여백 충분.

하단에 캡션 한 줄: "{{업무명}} — {{Q6 매핑}} 위에 얹는 자동화"
```

호출 결과 PNG가 떨어지면 "도식이 같은 폴더에 떨어졌습니다" 안내.

## 다음 단계 안내 (도식 생성 직후)

도식이 나오면 사용자에게 한 줄 더:

> **이 도식이 본인의 다음 7일 출발점입니다.** 오늘은 구상까지. 강의 후 GitHub workshop README의 "Day 7~" 항목을 참고해, 4축 중 본인이 고른 칸 하나를 실제로 얹어 보세요. 막히면 이 인터뷰 결과 표를 클로드에게 다시 던지며 *"이 워크플로의 ②번 처리 단계만 스킬로 만들어 줘"*라고 한 줄로 부탁하시면 됩니다.

## 사전 요구사항

- `/paperbanana` 스킬 (워크숍 셋팅에 포함)
- Gemini API 키 — 강의 당일 강사가 일회용 키를 공유 → `~/.claude/apikeys/gemini`에 저장하면 paperbanana가 자동 사용

## 메타 — 이 스킬도 본인이 손볼 수 있다

이 SKILL.md 자체도 한 장의 파일이다. 인터뷰 진행이 본인 분야에 맞지 않으면, 작업 끝에 클로드에게 한 줄 던지면 된다:

> *"오늘 작업 내용을 바탕으로 이 스킬(`~/.claude/skills/나의에이전트하네스/SKILL.md`)을 업데이트해 줘. Q3 처리 단계를 우리 학교 업무 흐름에 맞게 더 구체적인 보기로 바꿔 주고, Q6 4축 매핑도 본인 분야 예시 한두 개 추가."*

이게 4교시의 메타 메시지다 — **강사 스킬도 본인이 손보고, 손본 결과를 다시 스킬에 누적한다.**
